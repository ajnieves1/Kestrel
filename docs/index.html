How it works

These steps happen during one mission.

  1. The mission planner builds a survey path around the structure.
  2. The flight commander flies the copter to each waypoint.
  3. The defect detector reads the camera feed during the whole flight.
  4. On a detection, the mission director pauses the survey and flies a closer orbit around the defect.
  5. The system stores each photo and each detection.
  6. After landing, the report writer writes a report in markdown format.

Detector model

The corrosion detector is a YOLOv8n model, trained on the public corrosion-bi3q3 data set (CC BY 4.0) and deployed with ONNX Runtime. The script scripts/eval_model.py scores the shipped model against the held out test split through the same inference path that the flight stack uses. The test split played no part in training or in model selection.

Talk to it

The flight stack also works as an MCP server. An LLM client can fly the simulation through this server. The client can ask for telemetry data. The client can send a takeoff command. The client can send the copter to a position. The client can send a land command. Every command goes through the same guarded services that the autonomous mission uses. The safety guard node keeps final control at all times.

See the README for setup steps and the full tool list.
